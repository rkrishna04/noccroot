WEBSITE WORKFLOW - Complete User Journey

WORKFLOW 1: NEW USER ONBOARDING

Step 1: Root User Creates Super Admin

Root User Login
    ↓
Navigate to "Root User Management"
    ↓
Click "Create New Root User" (if needed) or "Manage Super Admins"
    ↓
Enter Details:
    - Full Name
    - Email
    - Temporary Password
    ↓
System sends welcome email with:
    - Temporary password
    - Login link
    - Initial setup instructions
    ↓
New Super Admin receives email & first login


Step 2: Super Admin Creates Normal Users & Admins

Super Admin Login
    ↓
Navigate to "User Management" Section
    ↓
Choose: Create Normal User OR Create Admin
    ↓
IF CREATE NORMAL USER:
    - Enter: Name, Email, Phone, Department, Post
    - System auto-assigns: "View Data Only" permission
    - Set temporary password
    ↓
IF CREATE ADMIN:
    - Enter: Name, Email, Phone, Department
    - System auto-assigns: View, Modify, Generate Report permissions
    - Set temporary password
    ↓
System sends welcome email to new user
    ↓
New user receives email & sets permanent password on first login




WORKFLOW 2: FIRST TIME USER LOGIN


User visits website
    ↓
Sees Login Page:
    ┌─────────────────────────────────┐
    │  N-DAMS Login                   │
    │                                 │
    │  Email: _________________       │
    │  Password: ______________       │
    │  [ ] Remember me                │
    │                                 │
    │  [    LOGIN    ]                │
    │                                 │
    │  Forgot Password? | Sign Up     │
    └─────────────────────────────────┘
    ↓
Enter Email & Password
    ↓
System validates credentials against database
    ↓
IF LOGIN SUCCESSFUL:
    - Check user role (Normal User/Admin/Super Admin/Root User)
    - Generate JWT token
    - Store session in database
    - Log IP address, device type, timestamp
    ↓
REDIRECT TO ROLE-SPECIFIC DASHBOARD:
    - Normal User → View-Only Dashboard
    - Admin → Admin Dashboard with monitoring tools
    - Super Admin → Management Dashboard with menu manager
    - Root User → System Administration Dashboard
    ↓
IF LOGIN FAILED (3+ attempts):
    - Show error message
    - After 3 failed attempts, lock account for 30 minutes
    - Send security alert email to user
    ↓
User sees dashboard based on their role
    ↓
System displays:
    - Welcome message: "Hello, [User Name]"
    - Last login time
    - Available modules in sidebar (based on permissions)
    - Quick stats/notifications
```

## 📱 KEY USER ACTIONS & SCREEN FLOWS

### Common User Actions

| User Action | Screen Flow | Time Required | Impact |
|---|---|---|---|
| **Login** | Enter credentials → Validate → Redirect to dashboard | 2-3 sec | Session created, logged |
| **View Data** | Click module → Select filters → Apply → See results | 1-2 sec | Data loaded, displayed |

## 📊 DATA FLOW - From Input to Audit Log

```
USER MODIFIES A RECORD
    ↓
USER FILLS FORM & CLICKS "SAVE"
    ├─ Frontend: Client-side validation
    │   - Check required fields
    │   - Validate data types
    │   - Check length/format
    ├─ If validation fails: Show error to user
    ├─ If validation passes: Send to backend
    ↓
BACKEND RECEIVES REQUEST
    ├─ Verify JWT token (still valid?)
    ├─ Verify user has "Modify Data" permission
    ├─ Verify user is in allowed time window (if time-limited)
    ├─ If checks fail: Reject with error
    ├─ If checks pass: Proceed to data update
    ↓
DATABASE UPDATE
    ├─ Read old record from database
    ├─ Update with new values
    ├─ Write new record to database
    ├─ Transaction commits
    ↓
AUDIT LOG CREATION (Automatic)
    ├─ Capture:
    │   - User ID
    │   - User Name
    │   - Timestamp (date + time + timezone)
    │   - IP Address
    │   - Device Type
    │   - Session ID
    │   - Table Name (which table modified)
    │   - Record ID (which row)
    │   - Field Name (which columns changed)
    │   - Old Value → New Value
    │   - Change Type (Add/Modify/Delete)
    │   - Reason (if provided by user)
    │
    ├─ Insert record into Modification_Logs table
    ↓
NOTIFICATION SENT TO USER
    ├─ Frontend: Show success message
    │   "Record updated successfully"
    │   "Changes logged at 3:45 PM"
    ↓
OPTIONAL: REPORT AUDIT LOG
    └─ Super Admin/Root User can view anytime
       - Who changed what
       - When exactly it happened
       - From which IP address
       - Using which device
       - Old values vs new values
```
